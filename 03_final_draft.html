This categorizes the mediums of public artworks in the art_sf dataset into predefined categories such as “Metal,” “Composite or Mixed Media,” “Stone,” and more. Firstly to create a dictionary medium_categories that lists different mediums associated with each category. Then, a custom function categorize_medium() is defined, which takes a medium as input and returns the corresponding category by checking which category list the medium belongs to.

The function is applied to the medium column of art_sf using sapply() inside a mutate() call from dplyr, creating a new column medium_category that holds the categorized medium for each artwork.

# Categorize a medium
categorize_medium <- function(medium) {
  for (category in names(medium_categories)) {
    if (medium %in% medium_categories[[category]]) {
      return(category)
    }
  }
  return("Other")
}

  mutate(medium_category = sapply(medium, categorize_medium))

#The map is created using the leaflet package represents the geographical distribution of public artworks in Maryland, categorized by their visibility. The map visually represents the distribution and visibility of public artworks in Maryland. Each image shall be marked on the map in its respective location with a colour indicating its visibility category. The map is a useful tool for exploring the visibility and geographic spread of public art in the region, thanks to interactive elements such as Popups and Legend Paintings, which provide additional context and information about each work.

library(leaflet)
library(dplyr)
library(sf)

tile_provider_url <- "https://server.arcgisonline.com/ArcGIS/rest/services/World_Topo_Map/MapServer/tile/{z}/{y}/{x}"

# Create the interactive map with modified basemap and dimensions
leaflet(data = art_sf, height=600, width=600) %>%
  addTiles(urlTemplate = tile_provider_url) %>%
  addCircleMarkers(
    ~longitude, ~latitude,
    color = ~case_when(
      visibility_category == "visible" ~ "red",
      visibility_category == "limited visibility" ~ "blue",
      visibility_category == "not visible" ~ "orange"
    ),
    popup = ~paste("Medium: ", medium, "<br>",
                   "Visibility: ", visibility_category),
    radius = 1.5
  ) %>%
  addLegend(
    position = "topleft",
    colors = c("red", "blue", "orange"),
    labels = c("Visible", "Limited Visibility", "Not Visible"),
    title = "Visibility"
  ) %>%
  addControl(
    html = "<h2>INTERACTIVE MAP OF ARTWORKS BY VISIBILITY CATEGORY.</h2>",
    position = "topright",
    className = "map-title"
  )

To overcome the overlapping of artworks on the map, I employed the clustering options. When the markers on the map close within a distance threshold, the Leaflet groups them into a cluster. The cluster is marked by a number showing the count of markers in the cluster. As you keep zooming into the map, the clusters divide to show smaller clusters and further show individual artworks in the map.

library(leaflet)
library(dplyr)
library(sf)

# Specify the URL template for the tile provider
tile_provider_url <- "https://server.arcgisonline.com/ArcGIS/rest/services/World_Topo_Map/MapServer/tile/{z}/{y}/{x}"

# Create the interactive map with modified basemap and dimensions
leaflet(data = art_sf, width = 600, height = 600) %>%
  addTiles(urlTemplate = tile_provider_url) %>%
  addCircleMarkers(
    ~longitude, ~latitude,
    color = ~case_when(
      visibility_category == "visible" ~ "red",
      visibility_category == "limited visibility" ~ "blue",
      visibility_category == "not visible" ~ "orange"
    ),
    popup = ~paste("Medium: ", medium, "<br>",
                   "Visibility: ", visibility_category),
    radius = 1.5,
    clusterOptions = markerClusterOptions()
  ) %>%
  addLegend(
    position = "bottomright",
    colors = c("red", "blue", "orange"),
    labels = c("Visible", "Limited Visibility", "Not Visible"),
    title = "Visibility"
  ) %>%
  addControl(
